# 🎨 GoF 디자인 패턴 핵심 요약 (정보처리기사 실기 대비)

## 🧩 디자인 패턴이란?
- 객체지향 설계의 **재사용성**, **유연성**, **유지보수성**을 높이기 위한 설계 방법.
- GoF(Gang of Four) 4인이 정리한 **23가지 패턴**으로 구성.
- 크게 **생성(Creational)** / **구조(Structural)** / **행위(Behavioral)** 패턴으로 구분됨.

---

## 🧱 1️⃣ 생성 패턴 (Creational Patterns) — 객체 생성 방식
> **무엇을 어떻게 생성할 것인가?**

| 패턴명 | 키워드 | 설명 요약 |
|:--|:--|:--|
| **Singleton** | 객체 하나만 생성 | 시스템 전체에서 단 하나의 인스턴스만 존재 |
| **Factory Method** | 객체 생성 전담 | 하위 클래스가 인스턴스 생성 방식 결정 |
| **Abstract Factory** | 관련 객체 집합 생성 | 관련된 객체들을 통일된 방식으로 생성 |
| **Builder** | 단계별 조립 | 복잡한 객체를 단계별로 조립 (생성과 표현 분리) |
| **Prototype** | 복제(복사) | 기존 객체를 복제해 새로운 객체 생성 (clone) |

> 💡 **암기 팁:** “**싱팩추빌프**”  
> 싱글톤 - 팩토리 - 추상 팩토리 - 빌더 - 프로토타입

---

## 🧩 2️⃣ 구조 패턴 (Structural Patterns) — 클래스/객체의 조합
> **객체나 클래스를 조합해 더 큰 구조를 만든다.**

| 패턴명 | 키워드 | 설명 요약 |
|:--|:--|:--|
| **Adapter** | 인터페이스 변환 | 호환되지 않는 인터페이스를 연결 |
| **Bridge** | 구현부와 추상부 분리 | 독립적으로 확장 가능 |
| **Composite** | 트리 구조 | 전체-부분(계층) 관계 표현 |
| **Decorator** | 기능 덧붙이기 | 기존 객체에 동적으로 기능 추가 |
| **Facade** | 통합 인터페이스 | 복잡한 시스템을 단순하게 표현 |
| **Flyweight** | 메모리 절약 | 공통 속성 공유 (경량 객체) |
| **Proxy** | 대리자 | 실제 객체에 대한 접근 제어 (대리 처리) |

> 💡 **암기 팁:** “**어브컴데퍼플프**”  
> 어댑터 - 브릿지 - 컴포지트 - 데코레이터 - 퍼사드 - 플라이웨이트 - 프록시

---

## ⚙️ 3️⃣ 행위 패턴 (Behavioral Patterns) — 객체 간 행위와 책임 분배
> **객체들이 어떻게 협력하는가?**

| 패턴명 | 키워드 | 설명 요약 |
|:--|:--|:--|
| **Chain of Responsibility** | 책임 연쇄 | 요청을 처리할 객체를 연결해 순차 처리 |
| **Command** | 명령 캡슐화 | 요청을 객체로 캡슐화 (되돌리기 등) |
| **Interpreter** | 문법 해석 | 언어의 문법을 클래스 구조로 표현 |
| **Iterator** | 순차 접근 | 내부 구조 몰라도 순차 탐색 가능 |
| **Mediator** | 중재자 | 객체 간 통신을 중재자에게 위임 |
| **Memento** | 상태 저장 | 객체 상태를 저장하고 복원 |
| **Observer** | 통보 | 상태 변화 시 관련 객체에 자동 통보 |
| **State** | 상태 전이 | 상태 변화에 따라 행동 변경 |
| **Strategy** | 알고리즘 교체 | 알고리즘을 캡슐화하여 교체 가능 |
| **Template Method** | 기본 알고리즘 틀 | 알고리즘의 구조는 상위 클래스가 정의 |
| **Visitor** | 새로운 연산 추가 | 구조는 고정, 연산만 확장 |

> 💡 **암기 팁:** “**책커인터이 미메옵스스트템비**”  
> 책임연쇄 - 커맨드 - 인터프리터 - 이터레이터 - 미디에이터 - 메멘토 - 옵서버 - 스테이트 - 스트래티지 - 템플릿 - 비지터

---

| 구분 | 개수 | 외우기 |

# 🧩 UML 핵심 암기 요약 (정보처리기사 실기 대비)

## 📘 UML 기본 개념
| 구분 | 내용 |
|:--|:--|
| **정의** | 객체지향 시스템을 시각화·명세화·구축·문서화하기 위한 표준 모델링 언어 |
| **구성요소** | **사물(Thing)**, **관계(Relationship)**, **다이어그램(Diagram)** |

---

## 🧱 1️⃣ 사물 (Things)
> UML의 기본 단위 요소

| 종류 | 키워드 | 설명 |
|:--|:--|:--|
| **구조 사물 (Structural)** | 클래스, 인터페이스, 컴포넌트, 노드 | 시스템의 **정적인 요소** |
| **행동 사물 (Behavioral)** | 상호작용, 상태머신 | **행동이나 동작** 표현 |
| **그룹 사물 (Grouping)** | 패키지 | 요소들을 **묶음 단위**로 표현 |
| **주석 사물 (Annotation)** | 노트 | 모델 요소에 **부가 설명** 추가 |

---

## 🔗 2️⃣ 관계 (Relationships)
> 사물 간의 연결

| 관계 | 키워드 | 설명 | 표기 |
|:--|:--|:--|:--:|
| **연관 (Association)** | 포함, 사용 | 클래스 간의 관계 (양방향/단방향) | ─────▶ |
| **집합 (Aggregation)** | 포함(약한 소유) | 전체-부분 관계 (부분 독립 존재) | ◇──▶ |
| **구성 (Composition)** | 포함(강한 소유) | 전체가 없어지면 부분도 소멸 | ◆──▶ |
| **일반화 (Generalization)** | 상속 | 상위 클래스의 속성과 연산 상속 | ▲──▶ |
| **의존 (Dependency)** | 일시적 의존 | 한 클래스가 다른 클래스를 일시적으로 참조 | ────▷ |

> 💡 **암기 포인트:**  
> 연관(선), 집합(빈 다이아), 구성(채워진 다이아), 일반화(삼각형), 의존(점선)

---

## 📊 3️⃣ 다이어그램 (Diagram)
> UML 다이어그램은 **정적(구조)** 과 **동적(행동)** 으로 나뉨

| 구분 | 다이어그램 종류 | 설명 |
|:--|:--|:--|
| **정적 다이어그램 (Structural)** | 클래스, 객체, 컴포넌트, 배치, 패키지 | 시스템의 **구조** 표현 |
| **동적 다이어그램 (Behavioral)** | 유스케이스, 시퀀스, 커뮤니케이션, 상태, 활동 | 시스템의 **동작 흐름** 표현 |

---

## 🧩 4️⃣ 클래스 다이어그램 (Class Diagram)
> 💥 **실기 핵심 출제 포인트!**

| 요소 | 키워드 | 설명 |
|:--|:--|:--|
| **클래스** | 속성 + 메서드 | 객체 설계의 기본 단위 |
| **속성(Attribute)** | 멤버변수 | `이름 : 타입` 형식 |
| **연산(Operation)** | 메서드 | `메서드명(매개변수) : 반환형` |
| **가시성(접근제한자)** | `+` public / `-` private / `#` protected | UML 표기법 |
| **관계선** | 연관, 상속, 집합, 구성 등 | 관계 종류별로 선 모양 다름 |

# 🔐 정보처리기사 실기 대비 — 네트워크 보안 핵심 요약

## 📘 1️⃣ 네트워크 공격 (Network Attack)

| 공격 유형 | 키워드 | 설명 |
|:--|:--|:--|
| **스니핑 (Sniffing)** | 도청 | 네트워크 트래픽을 가로채 데이터(패킷) 탈취 |
| **스푸핑 (Spoofing)** | 위조 | IP, ARP, DNS 등을 위조하여 속이는 공격 |
| **ARP 스푸핑** | MAC 주소 위조 | ARP 테이블을 조작해 중간자 위치 확보 |
| **IP 스푸핑** | IP 주소 위조 | 신뢰 관계를 악용한 접근 |
| **DNS 스푸핑** | 가짜 사이트 유도 | DNS 정보 변조로 사용자를 피싱 사이트로 유도 |
| **DDoS (Distributed Denial of Service)** | 분산 서비스 거부 | 다수의 좀비 PC로 서버 과부하 유발 |
| **Ping of Death / Smurf** | ICMP 기반 공격 | 과도한 패킷 전송으로 시스템 다운 유도 |
| **세션 하이재킹 (Session Hijacking)** | 세션 탈취 | 인증된 사용자 세션을 가로채 불법 접근 |
| **피싱 / 파밍** | 사회공학적 공격 | 사용자를 속여 개인정보 탈취 |

> 💡 **암기 포인트:**  
> 스니핑(도청), 스푸핑(위조), DDoS(마비), 하이재킹(세션탈취)

## 🧱 2️⃣ 접근 통제 (Access Control)
> 사용자가 **시스템이나 자원에 접근 가능한지 여부를 결정**하는 방식

| 유형 | 키워드 | 설명 |
|:--|:--|:--|
| **DAC (Discretionary Access Control)** | 임의적 접근 통제 | 사용자가 접근 권한을 **임의로 부여/회수** 가능 |
| **MAC (Mandatory Access Control)** | 강제적 접근 통제 | **보안 등급** 기반, 관리자가 강제로 제어 |
| **RBAC (Role-Based Access Control)** | 역할 기반 접근 통제 | **역할(Role)**에 따라 권한을 부여 |

> 💡 **암기법:**  
> DAC(사용자 중심), MAC(관리자 중심), RBAC(역할 중심)

---

## 🧩 3️⃣ 인증 기술 (Authentication)
> 사용자가 **누구인지 확인하는 과정**

| 기술 | 키워드 | 설명 |
|:--|:--|:--|
| **OTP (One-Time Password)** | 1회용 비밀번호 | 매 로그인 시마다 새로운 비밀번호 생성 |
| **CHAP (Challenge Handshake Authentication Protocol)** | 챌린지 응답 방식 | 서버가 임의 난수를 보내 인증 확인 |
| **EAP (Extensible Authentication Protocol)** | 확장형 인증 프레임워크 | 무선 네트워크(Wi-Fi 등)에서 자주 사용 |
| **Kerberos** | 티켓 기반 인증 | 대칭키 기반, 중앙 인증 서버를 통한 티켓 교환 |
| **SSO (Single Sign-On)** | 한번 로그인으로 전체 접근 | 여러 시스템을 하나의 인증으로 접근 가능 |
| **Biometric** | 생체인증 | 지문, 홍채, 얼굴, 음성 등 고유 정보 기반 |

> 💡 **암기 포인트:**  
> OTP(1회용), CHAP(질문·응답), Kerberos(티켓), SSO(한번 로그인)

## 🔐 4️⃣ 암호화 기법 (Cryptography)

### 🔸 분류 개념
| 구분 | 설명 | 예시 |
|:--|:--|:--|
| **양방향 암호화 (복호화 가능)** | 암호화된 데이터를 복호화 가능 | 대칭키 / 비대칭키 |
| **단방향 암호화 (복호화 불가)** | 복호화 불가능, 해시함수 형태 | 비밀번호 저장용 |

---

### 🔸 (1) 대칭키 암호화 (Symmetric Key Encryption)
> 암호화와 복호화에 **같은 키** 사용

| 알고리즘 | 키워드 | 설명 |
|:--|:--|:--|
| **DES** | 56비트 블록 암호 | 오래된 표준 대칭키 알고리즘 |
| **3DES** | 3중 DES | DES를 3번 적용해 보안 강화 |
| **AES** | 고급 암호 표준 | 현재 가장 널리 사용되는 대칭키 |
| **SEED** | 한국형 대칭키 | KISA 개발, 국내 금융권 표준 |
| **ARIA** | 국가 암호화 표준 | 공공기관용 국산 알고리즘 |

> 💡 키워드 암기: **“같은 키, 빠르지만 키 관리 어려움”**

---

### 🔸 (2) 비대칭키 암호화 (Asymmetric Key Encryption)
> 암호화와 복호화에 **다른 키 (공개키/개인키)** 사용

| 알고리즘 | 키워드 | 설명 |
|:--|:--|:--|
| **RSA** | 공개키 방식 | 큰 소수의 곱을 이용한 암호화 |
| **ECC (Elliptic Curve Cryptography)** | 타원곡선 | 모바일 환경에서 효율적 |
| **DSA (Digital Signature Algorithm)** | 전자서명용 | 서명과 검증용 알고리즘 |

> 💡 키워드 암기: **“다른 키, 안전하지만 느림”**

---

### 🔸 (3) 단방향 암호화 (One-Way Hash)
> 복호화 불가능한 **해시(Hash)** 기반 암호화

| 알고리즘 | 키워드 | 설명 |
|:--|:--|:--|
| **MD5** | 128비트 해시 | 충돌 발생 위험 (보안 낮음) |
| **SHA-1 / SHA-256** | 보안 해시 표준 | 현재 SHA-2 계열 사용 권장 |
| **HAS-160** | 한국형 해시 | 국내 표준, 전자서명용 |
| **PBKDF2, bcrypt, scrypt** | 비밀번호 해시 강화 | 반복 계산으로 무차별 대입 공격 방지 |

> 💡 키워드 암기: **“단방향 → 복호화 불가 → 무결성 검증용”**

## 🧠 전체 구조 요약
